## 3. Оптимизация отображения фото основателя

**Затронутые файлы:**
- `src/pages/ru/HomeRu.tsx`
- `src/pages/en/HomeEn.tsx`
- `src/pages/ru/AboutRu.tsx`
- `src/pages/en/AboutEn.tsx`

**Изменения:**
- Изменить соотношение сторон с `aspect-[4/5]` на `aspect-[3/4]` — это делает фото менее "обрезанным"
- Добавить `object-position: top` — фокус на верхнюю часть изображения
- Уменьшить масштаб через `scale-90` в CSS — визуально "отодвигает" изображение

Это создаст более нейтральную, комфортную композицию без эффекта "слишком близко".

## 4-6. Настройка favicon как статического бренд-актива

### 4.1 Создание набора favicon-файлов

Нужно создать в директории `public/`:
- `/favicon.ico` — уже существует, проверить/обновить
- `/favicon-32x32.png` — новый файл (логотип V на синем фоне)
- `/favicon-16x16.png` — новый файл (логотип V на синем фоне)
- `/apple-touch-icon.png` (180x180) — новый файл (логотип V на синем фоне)

**Примечание:** Поскольку у нас есть только `V-3.png` (загруженный логотип), мы:
1. Скопируем его как базовый favicon.png
2. Создадим ссылки на него в HTML с правильными именами

### 4.2 Обновление index.html

**Файл:** `index.html`

Заменить текущие строки с favicon на:
```html
<link rel="icon" href="/favicon.ico" sizes="any">
<link rel="icon" type="image/png" sizes="32x32" href="/favicon-32x32.png">
<link rel="icon" type="image/png" sizes="16x16" href="/favicon-16x16.png">
<link rel="apple-touch-icon" sizes="180x180" href="/apple-touch-icon.png">
```

### 4.3 Отключение автозамены Lovable

Lovable не имеет механизма автозамены favicon, если файлы явно указаны в `index.html` и размещены в `public/`. Достаточно:
- Явно прописать все ссылки в `<head>`
- Разместить файлы в `public/`
- Не использовать динамические иконки
